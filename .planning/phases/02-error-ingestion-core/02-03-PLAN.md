**server/src/ingest/scrubber.mpl** -- PII scrubbing pipeline.

Locked decision: Scrub at ingestion time, PII never hits disk unscrubbed. Replace with `[Filtered]` (Sentry convention).

**Discovery first:** Check if Mesh has regex support:
- Query GitNexus for "regex", "Regex", "match", "pattern" in the mesh repo
- If regex exists: use it for IP, email, credit card patterns
- If no regex: use String.contains / String.replace with literal patterns (less precise but functional)

**Default scrub rules (hardcoded):**
1. IPv4 addresses: replace patterns like `123.456.789.012` with `[Filtered]`
2. Email patterns: replace `user@domain.com` patterns with `[Filtered]`
3. Credit card-like patterns: 16-digit sequences with optional separators
4. Authorization header values: replace entirely with `[Filtered]`
5. Cookie values: replace entirely with `[Filtered]`

**server/src/ingest/fingerprint.mpl** -- Event fingerprinting algorithm.

Per requirement ERR-02: exception type + normalized top 3-5 application stack frames (line numbers stripped, framework frames excluded).

```
# Compute a fingerprint hash for event deduplication.
# Input: exception_type and stacktrace_json (the JSON string of frames array)
# Output: SHA-256 hex string

pub fn compute_fingerprint(exception_type :: String, stacktrace_json :: String) -> String do
  # 1. Parse stacktrace_json into frame list (Json.parse)
  # 2. Filter to in_app frames only:
  #    - Sentry format: frame has "in_app" == true
  #    - OTLP/Generic: frame filename does NOT contain "node_modules/"
  # 3. Take top 5 frames (end of array = closest to error in Sentry format)
  #    Use List.reverse to get most recent first, then List.take(frames, 5)
  # 4. For each frame, normalize: filename + ":" + function_name
  #    Strip line numbers (lineno) and column numbers (colno) entirely
  # 5. Concatenate: exception_type ++ "|" ++ frame1 ++ "|" ++ frame2 ++ ...
  # 6. Return Crypto.sha256(concatenated_string)
end
```

If stacktrace_json is empty/unparseable or has no app frames, fall back to fingerprinting on exception_type + message (less precise but prevents crashes):
```
fn fallback_fingerprint(exception_type :: String, message :: String) -> String do
  Crypto.sha256(exception_type <> "|" <> message)
end
```

**Important:** The fingerprint must be deterministic -- same input always produces same output. Do NOT include timestamps, event_id, or any per-event unique data.
