  <action>
Create a new migration file that creates 4 tables in FK dependency order:

1. **issues table** (no FK to events -- events FK to issues):
```sql
CREATE TABLE issues (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  fingerprint TEXT NOT NULL,
  title TEXT NOT NULL,
  level TEXT NOT NULL DEFAULT 'error',
  status TEXT NOT NULL DEFAULT 'open',
  first_seen TIMESTAMPTZ NOT NULL DEFAULT now(),
  last_seen TIMESTAMPTZ NOT NULL DEFAULT now(),
  event_count INT NOT NULL DEFAULT 1,
  environment TEXT,
  metadata_json TEXT DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(project_id, fingerprint)
);
```
Use Pool.execute for this table (needs composite UNIQUE constraint, same pattern as org_memberships).
Add indexes: `idx_issues_project_status (project_id, status)`, `idx_issues_project_last_seen (project_id, last_seen DESC)`, `idx_issues_fingerprint (project_id, fingerprint)`.

2. **events table** (FK to projects and issues):
Use Pool.execute (not Migration.create_table) since we need the hypertable conversion immediately after.
```sql
CREATE TABLE events (
  id UUID DEFAULT gen_random_uuid(),
  project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  issue_id UUID NOT NULL REFERENCES issues(id),
  event_id TEXT NOT NULL,
  timestamp TIMESTAMPTZ NOT NULL DEFAULT now(),
  received_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  platform TEXT NOT NULL DEFAULT 'unknown',
  level TEXT NOT NULL DEFAULT 'error',
  message TEXT,
  exception_type TEXT,
  exception_value TEXT,
  stacktrace_json TEXT,
  environment TEXT NOT NULL DEFAULT 'production',
  release_tag TEXT,
  server_name TEXT,
  tags_json TEXT DEFAULT '{}',
  extra_json TEXT DEFAULT '{}',
  contexts_json TEXT DEFAULT '{}',
  sdk_name TEXT,
  sdk_version TEXT,
  fingerprint TEXT NOT NULL
);
```
IMPORTANT: Do NOT include PRIMARY KEY on id for the events table -- TimescaleDB hypertables require the partitioning column (timestamp) to be part of any unique index. Use a separate unique index on (id, timestamp) instead, or omit the PK entirely (id is still generated, just not a PK constraint).

Then convert to hypertable: `SELECT create_hypertable('events', 'timestamp');`

Add indexes: `idx_events_project_ts (project_id, timestamp DESC)`, `idx_events_issue (issue_id, timestamp DESC)`, `idx_events_fingerprint (fingerprint)`, `idx_events_environment (project_id, environment)`, `idx_events_level (project_id, level)`.

3. **rate_limit_configs table**:
Use Migration.create_table:
```
"id:UUID:PRIMARY KEY DEFAULT gen_random_uuid()",
"org_id:UUID:NOT NULL REFERENCES organizations(id) ON DELETE CASCADE UNIQUE",
"events_per_minute:INT:NOT NULL DEFAULT 1000",
"burst_limit:INT:NOT NULL DEFAULT 100",
"created_at:TIMESTAMPTZ:NOT NULL DEFAULT now()",
"updated_at:TIMESTAMPTZ:NOT NULL DEFAULT now()"
```

4. **scrub_rules table**:
Use Migration.create_table:
```
"id:UUID:PRIMARY KEY DEFAULT gen_random_uuid()",
"org_id:UUID:NOT NULL REFERENCES organizations(id) ON DELETE CASCADE",
"pattern:TEXT:NOT NULL",
"replacement:TEXT:NOT NULL DEFAULT '[Filtered]'",
"description:TEXT",
"is_active:BOOLEAN:NOT NULL DEFAULT true",
"created_at:TIMESTAMPTZ:NOT NULL DEFAULT now()"
```
Add index: `idx_scrub_rules_org ON scrub_rules (org_id) WHERE is_active = true` (via Pool.execute).

The `down` function drops tables in reverse order: scrub_rules, rate_limit_configs, events, issues.

Follow established patterns: `pub fn up(pool :: PoolHandle) -> Int!String`, use `?` operator throughout, `Ok(0)` at end.
