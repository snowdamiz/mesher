<!-- Backend API contracts from Plans 03-05 -->

Auth API:
- POST /api/auth/register {email, password} -> 201 {id, email} + Set-Cookie
- POST /api/auth/login {email, password} -> 200 {id, email} + Set-Cookie
- POST /api/auth/logout -> 200 {status} + clear cookie
- GET /api/auth/me -> 200 {id, email} (with session cookie)
- POST /api/auth/reset-password {email} -> 200 {status}
- POST /api/auth/reset-password/confirm {token, new_password} -> 200 {status}
- GET /api/auth/oauth/google -> 302 redirect to Google (SaaS only)

Org API:
- POST /api/orgs {name} -> 201 {id, name, schema_name}
- GET /api/orgs -> 200 {organizations: [...]}
- GET /api/orgs/:id -> 200 {id, name, ...}

Invite API:
- POST /api/orgs/:id/invites {email} -> 201 {id, email, expires_at}
- GET /api/orgs/:id/invites -> 200 {invites: [...]}
- DELETE /api/orgs/:id/invites/:iid -> 200
- POST /api/invites/:token/accept -> 200 {status, org_id}

Project API:
- POST /api/orgs/:id/projects {name} -> 201 {id, name}
- GET /api/orgs/:id/projects -> 200 {projects: [...]}

API Key API:
- POST /api/orgs/:id/projects/:pid/api-keys {label?} -> 201 {id, key, prefix, dsn}
- GET /api/orgs/:id/projects/:pid/api-keys -> 200 {api_keys: [...]}
- POST /api/orgs/:id/api-keys/:kid/revoke -> 200

Streem-2 patterns (from skills/components + skills/lit):
- signal(initial) for reactive state
- effect(() => { ... }) for side effects
- Show component for conditional rendering
- on:event for LitUI component events
- prop:data for object properties (not HTML attributes)

**frontend/src/lib/api.ts** — Centralized API client:
```typescript
async function request(method: string, path: string, body?: unknown) {
  const opts: RequestInit = {
    method,
    credentials: 'include', // send cookies
    headers: body ? { 'Content-Type': 'application/json' } : {},
    body: body ? JSON.stringify(body) : undefined,
  }
  const res = await fetch(path, opts)
  const data = await res.json().catch(() => ({}))
  if (!res.ok) throw { status: res.status, ...data }
  return data
}

export const api = {
  auth: {
    register: (email: string, password: string) => request('POST', '/api/auth/register', { email, password }),
    login: (email: string, password: string) => request('POST', '/api/auth/login', { email, password }),
    logout: () => request('POST', '/api/auth/logout'),
    me: () => request('GET', '/api/auth/me'),
    resetPassword: (email: string) => request('POST', '/api/auth/reset-password', { email }),
    confirmReset: (token: string, password: string) => request('POST', '/api/auth/reset-password/confirm', { token, new_password: password }),
  },
  orgs: {
    create: (name: string) => request('POST', '/api/orgs', { name }),
    list: () => request('GET', '/api/orgs'),
    get: (id: string) => request('GET', `/api/orgs/${id}`),
    invites: {
      create: (orgId: string, email: string) => request('POST', `/api/orgs/${orgId}/invites`, { email }),
      list: (orgId: string) => request('GET', `/api/orgs/${orgId}/invites`),
      revoke: (orgId: string, inviteId: string) => request('DELETE', `/api/orgs/${orgId}/invites/${inviteId}`),
      accept: (token: string) => request('POST', `/api/invites/${token}/accept`),
    },
    projects: {
      create: (orgId: string, name: string) => request('POST', `/api/orgs/${orgId}/projects`, { name }),
      list: (orgId: string) => request('GET', `/api/orgs/${orgId}/projects`),
    },
    apiKeys: {
      create: (orgId: string, projectId: string, label?: string) => request('POST', `/api/orgs/${orgId}/projects/${projectId}/api-keys`, { label }),
      list: (orgId: string, projectId: string) => request('GET', `/api/orgs/${orgId}/projects/${projectId}/api-keys`),
      revoke: (orgId: string, keyId: string) => request('POST', `/api/orgs/${orgId}/api-keys/${keyId}/revoke`),
    },
  },
}
```

- If logged in with orgs: redirect to org settings (subsequent per CONTEXT.md)
- Route to appropriate page component based on hash

Use LitUI form components throughout. Error states show inline via signals.
