**src/auth/reset.mpl** — Password reset flow (AUTH-04):

Per CONTEXT.md: Email link with time-limited token, expires 1 hour.

`fn request_reset_handler(request :: Request) -> Response` (POST /api/auth/reset-password):
1. Parse JSON body: extract `email`
2. Look up user: `SELECT id FROM users WHERE email = $1`
3. If not found: STILL return 200 (prevent email enumeration)
4. Generate token: `Crypto.uuid4()`
5. Hash token for storage: `Crypto.sha256(token)` (we store the hash, send the raw token in email)
6. Generate record ID: `Crypto.uuid4()`
7. Invalidate any existing tokens for this user: `UPDATE password_reset_tokens SET used_at = NOW() WHERE user_id = $1 AND used_at IS NULL`
8. INSERT INTO password_reset_tokens (id, user_id, token_hash, expires_at) VALUES ($1, $2, $3, NOW() + INTERVAL '1 hour')
9. Send email via `send_email(email, "Reset your Mesher password", "Click here to reset: {app_url}/reset-password?token={token}")`
10. Return `HTTP.response(200, json { status: "reset email sent" })`

`fn confirm_reset_handler(request :: Request) -> Response` (POST /api/auth/reset-password/confirm):
1. Parse JSON body: extract `token` and `new_password`
2. Hash the provided token: `Crypto.sha256(token)`
3. Query: `SELECT t.id, t.user_id FROM password_reset_tokens t WHERE t.token_hash = $1 AND t.expires_at > NOW() AND t.used_at IS NULL`
4. If not found: return `HTTP.response(400, json { error: "invalid or expired token" })`
5. Generate new salt: `Crypto.uuid4()`
6. Hash new password: `Crypto.hmac_sha256(Config.session_secret() <> new_salt, new_password)`
7. Update user: `UPDATE users SET password_hash = $1, password_salt = $2, updated_at = NOW() WHERE id = $3`
8. Mark token as used: `UPDATE password_reset_tokens SET used_at = NOW() WHERE id = $1`
9. Invalidate all existing sessions for this user: `Session.destroy_user_sessions(pool, user_id)` — forces re-login with new password
10. Return `HTTP.response(200, json { status: "password reset" })`

**src/auth/oauth.mpl** — Google OAuth (AUTH-05, SaaS only):

Per CONTEXT.md: SaaS Google OAuth — single button, first sign-in auto-creates account.

`fn google_oauth_start(request :: Request) -> Response` (GET /api/auth/oauth/google):
1. Check tier: if not SaaS, return 403
2. Generate state parameter: `Crypto.uuid4()` — store in a temporary record or cookie for CSRF protection (per Pitfall 4)
3. Build Google authorization URL:
   ```
   https://accounts.google.com/o/oauth2/v2/auth?
     client_id={Config.google_client_id()}&
     redirect_uri={app_url}/api/auth/oauth/google/callback&
     response_type=code&
     scope=openid email&
     state={state}&
     access_type=online
   ```
4. Return redirect response: `HTTP.response(302, "")` with Location header
   NOTE: If Mesh HTTP.response doesn't support custom headers for redirects, may need to return an HTML page with a meta refresh or JS redirect. Document this limitation if encountered.

`fn google_oauth_callback(request :: Request) -> Response` (GET /api/auth/oauth/google/callback):
1. Extract `code` and `state` from query parameters: `Request.query(request, "code")`, `Request.query(request, "state")`
2. Validate state matches stored state (CSRF check)
3. Exchange code for tokens: POST to `https://oauth2.googleapis.com/token` using `Http.build(:post, ...)` with:
   - client_id, client_secret, code, redirect_uri, grant_type=authorization_code
4. Parse token response: extract `id_token` (JWT) or `access_token`
5. Fetch user info: GET `https://www.googleapis.com/oauth2/v2/userinfo` with Authorization: Bearer {access_token}
6. Parse user info: extract `email`
7. Upsert user:
   - Check if user exists: `SELECT id FROM users WHERE email = $1`
   - If exists: use existing user_id
   - If not: INSERT new user with a random password_hash/salt (OAuth users don't use password login)
8. Create session: `Session.create_session(pool, user_id)`
9. Redirect to frontend with session cookie: `HTTP.response(302, "")` with Location: / and Set-Cookie header

**src/org/invite_handlers.mpl** — Invite flow (ORG-02, ORG-03):

Per CONTEXT.md: Owner sends email invite, 7-day validity, invitee with/without account can accept.

`fn create_invite_handler(request :: Request) -> Response` (POST /api/orgs/:org_id/invites):
1. Extract org_id from path param, user_id from session
2. Verify user is org owner: `SELECT role FROM org_memberships WHERE org_id = $1 AND user_id = $2`
3. If role != "owner": return 403
4. Parse JSON body: extract `email`
5. Check if already a member: `SELECT id FROM org_memberships m JOIN users u ON m.user_id = u.id WHERE m.org_id = $1 AND u.email = $2`
6. If already member: return `HTTP.response(409, json { error: "user is already a member" })`
7. Check for existing pending invite: `SELECT id FROM invites WHERE org_id = $1 AND email = $2 AND accepted_at IS NULL AND revoked_at IS NULL AND expires_at > NOW()`
8. If pending invite exists: return `HTTP.response(409, json { error: "invite already pending" })`
9. Generate invite token: `Crypto.uuid4()`
10. Generate invite ID: `Crypto.uuid4()`
11. INSERT INTO invites (id, org_id, email, token, invited_by, expires_at) VALUES ($1, $2, $3, $4, $5, NOW() + INTERVAL '7 days')
12. Send invite email: `send_email(email, "You've been invited to {org_name} on Mesher", "Accept invite: {app_url}/invites/{token}/accept")`
13. Return `HTTP.response(201, json { id: invite_id, email: email, expires_at: "..." })`

`fn accept_invite_handler(request :: Request) -> Response` (POST /api/invites/:token/accept):
1. Extract token from path param
2. Look up invite: `SELECT i.id, i.org_id, i.email FROM invites i WHERE i.token = $1 AND i.accepted_at IS NULL AND i.revoked_at IS NULL AND i.expires_at > NOW()`
3. If not found: return `HTTP.response(400, json { error: "invalid or expired invite" })`
4. Check if user is authenticated (has session cookie):
   a. If authenticated: verify their email matches invite email (or allow any authenticated user to accept — per CONTEXT, existing account users auto-join)
   b. If not authenticated: return `HTTP.response(401, json { error: "login or register first", redirect: "/register?invite={token}" })` — the frontend should redirect to registration with the invite token
5. If user exists and is authenticated:
   - Generate membership ID: `Crypto.uuid4()`
   - INSERT INTO org_memberships (id, org_id, user_id, role) VALUES ($1, $2, $3, 'member')
   - UPDATE invites SET accepted_at = NOW() WHERE id = $1
   - Return `HTTP.response(200, json { status: "joined", org_id: org_id })`

`fn revoke_invite_handler(request :: Request) -> Response` (DELETE /api/orgs/:org_id/invites/:invite_id):
1. Extract org_id, invite_id from path params, user_id from session
2. Verify owner role
3. UPDATE invites SET revoked_at = NOW() WHERE id = $1 AND org_id = $2 AND revoked_at IS NULL
4. Return 200 on success, 404 if invite not found

`fn list_invites_handler(request :: Request) -> Response` (GET /api/orgs/:org_id/invites):
1. Verify membership
2. Query: `SELECT id, email, token, expires_at, accepted_at, revoked_at, created_at FROM invites WHERE org_id = $1 ORDER BY created_at DESC`
3. Return invite list as JSON

**src/project/handlers.mpl** — Projects + API keys (ORG-04, ORG-05):

`fn create_project_handler(request :: Request) -> Response` (POST /api/orgs/:org_id/projects):
1. Extract org_id, user_id from session; verify membership
2. Parse JSON body: extract `name`
3. Validate: name 1-100 chars
4. Generate project ID: `Crypto.uuid4()`
5. Use `with_org_schema(pool, org_id, fn(conn) -> ...)`:
   - INSERT INTO projects (id, name) VALUES ($1, $2)
6. Return `HTTP.response(201, json { id: project_id, name: name })`

`fn list_projects_handler(request :: Request) -> Response` (GET /api/orgs/:org_id/projects):
1. Verify membership
2. Use `with_org_schema`: SELECT * FROM projects ORDER BY created_at
3. Return projects as JSON

`fn create_api_key_handler(request :: Request) -> Response` (POST /api/orgs/:org_id/projects/:project_id/api-keys):
1. Verify membership
2. Parse JSON body: optional `label`
3. Generate raw API key: `Crypto.uuid4()` — full key, shown once to user
4. Key prefix: first 8 characters of the key (for display identification)
5. Hash key for storage: `Crypto.sha256(raw_key)` — store hash, not raw key
6. Generate key ID: `Crypto.uuid4()`
7. Use `with_org_schema`:
   - INSERT INTO api_keys (id, project_id, key_hash, key_prefix, label) VALUES ($1, $2, $3, $4, $5)
8. Build DSN: `https://{key_prefix}:{raw_key}@{host}/api/{project_id}/`
9. Return `HTTP.response(201, json { id: key_id, key: raw_key, prefix: key_prefix, dsn: dsn })` — raw key shown ONLY here, never retrievable again

`fn revoke_api_key_handler(request :: Request) -> Response` (POST /api/orgs/:org_id/api-keys/:key_id/revoke):
1. Verify membership
2. Use `with_org_schema`:
   - UPDATE api_keys SET revoked_at = NOW() WHERE id = $1 AND revoked_at IS NULL
3. Return 200

`fn list_api_keys_handler(request :: Request) -> Response` (GET /api/orgs/:org_id/projects/:project_id/api-keys):
1. Verify membership
2. Use `with_org_schema`: SELECT id, project_id, key_prefix, label, created_at, revoked_at FROM api_keys WHERE project_id = $1 ORDER BY created_at DESC
3. Return keys (WITHOUT key_hash — only prefix for identification)
