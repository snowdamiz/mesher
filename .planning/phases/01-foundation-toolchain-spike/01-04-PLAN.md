**src/org/schema.mpl** — Schema provisioning (refined from tenant.mpl skeleton in Plan 01):

If `provision_org_schema` was already created in `src/db/tenant.mpl` in Plan 01, this file extends it or the handler calls tenant.mpl directly. If Plan 01 only created the function signature, implement it here:

`fn provision_org_schema(pool, org_id :: String) -> Int!String`:
1. Sanitize org_id: ensure it only contains alphanumeric chars and hyphens (strip or reject anything else to prevent SQL injection in schema name)
2. Replace hyphens with underscores for PG schema name compatibility: `let schema_name = "org_" <> String.replace(org_id, "-", "_")`
3. Execute: `CREATE SCHEMA IF NOT EXISTS {schema_name}` (use string interpolation — this is DDL, not parameterizable)
4. Within a transaction with `SET LOCAL search_path TO {schema_name}, public`:
   - CREATE TABLE IF NOT EXISTS projects (id TEXT PK, name TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW())
   - CREATE TABLE IF NOT EXISTS api_keys (id TEXT PK, project_id TEXT FK->projects, key_hash TEXT NOT NULL, key_prefix TEXT NOT NULL, label TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), revoked_at TIMESTAMPTZ)
5. Return Ok(0) on success

IMPORTANT: The schema name includes `public` in search_path so that PostgreSQL extension functions (TimescaleDB) remain accessible — per Pitfall 6 in research.

**src/org/handlers.mpl** — HTTP handlers:

`fn create_org_handler(request :: Request) -> Response` (POST /api/orgs):
1. Extract session and user_id from cookie (same pattern as auth/me handler — re-validate session, extract user_id)
2. Parse JSON body: extract `name` field
3. Validate: name is non-empty, length 1-100 chars
4. Generate org ID: `Crypto.uuid4()`
5. Generate schema name: `"org_" <> String.replace(org_id, "-", "_")`
6. Begin transaction on public schema:
   - INSERT INTO organizations (id, name, owner_id, schema_name) VALUES ($1, $2, $3, $4)
   - Generate membership ID: `Crypto.uuid4()`
   - INSERT INTO org_memberships (id, org_id, user_id, role) VALUES ($1, $2, $3, 'owner')
7. Provision org schema: call `provision_org_schema(pool, org_id)`
8. Return `HTTP.response(201, json { id: org_id, name: name, schema_name: schema_name })`
9. On any failure: roll back and return `HTTP.response(500, json { error: "failed to create organization" })`

`fn list_user_orgs_handler(request :: Request) -> Response` (GET /api/orgs):
1. Extract user_id from session
2. Query: `SELECT o.id, o.name, o.schema_name, o.created_at, m.role FROM organizations o JOIN org_memberships m ON o.id = m.org_id WHERE m.user_id = $1 ORDER BY o.created_at`
3. Return `HTTP.response(200, json { organizations: [...] })` — map rows to JSON

`fn get_org_handler(request :: Request) -> Response` (GET /api/orgs/:org_id):
1. Extract org_id from path param: `Request.param(request, "org_id")`
2. Extract user_id from session
3. Verify membership: `SELECT role FROM org_memberships WHERE org_id = $1 AND user_id = $2`
4. If not a member: return 403
5. Query org details: `SELECT * FROM organizations WHERE id = $1`
6. Return org details as JSON

**Update src/main.mpl** — Wire org routes (after auth_middleware):
- `HTTP.on_post(r, "/api/orgs", create_org_handler)`
- `HTTP.on_get(r, "/api/orgs", list_user_orgs_handler)`
- `HTTP.on_get(r, "/api/orgs/:org_id", get_org_handler)`
